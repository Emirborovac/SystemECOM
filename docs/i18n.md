# i18n conventions (SystemECOM)

## Locales

- **en** (default)
- **bs**
- **de**

## Frontend

- All UI text must use `next-intl` keys from `wlms-frontend/src/messages/{en,bs,de}.json`.
- Avoid hardcoded strings in pages/components (exceptions: IDs, debug labels, temporary placeholders).
- Locale is part of the URL: `/{locale}/...`.

- Templates should use **keys + params** (no hardcoded user-facing strings in route handlers).
